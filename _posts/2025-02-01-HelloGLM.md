## GLM 라이브러리: 수학적 변환을 OpenGL 코드로 적용하기

**GLM (OpenGL Mathematics)** 은 OpenGL 환경에서 벡터와 행렬 연산을 매우 직관적이고 쉽게 처리할 수 있도록 돕는 C++ 수학 헤더 전용 라이브러리다.

이번 포스트에서는 점 $(1,0,0)$을 $(1,1,0)$ 위치로 **이동(Translation)** 시키고 **회전(Rotation)** 하는 예제를 통해 GLM 함수의 기초적인 활용법을 다룬다. 먼저 이동 변환부터 살펴본다.

### 1. 이동 (Translation) 

이동(Translation)은 공간 상에서 객체의 위치 좌표를 물리적으로 옮기는 선형 대수학적 변환이다. GLM을 사용하면 단 몇 줄의 코드로 4x4 이동 변환 행렬을 생성하고 특정 벡터 위치에 적용할 수 있다.

수학적으로 이동 변환은 항등 행렬에 이동값 벡터를 조합하여 도출하며, 벡터와의 행렬 곱 형태로 계산된다.
$$
\begin{bmatrix}
1 & 0 & 0 & T_x \\
0 & 1 & 0 & T_y \\
0 & 0 & 1 & T_z \\
0 & 0 & 0 & 1
\end{bmatrix}
\cdot
\begin{bmatrix}
x \\ y \\ z \\ 1
\end{bmatrix}
=
\begin{bmatrix}
x + T_x \\
y + T_y \\
z + T_z \\
1
\end{bmatrix}
$$

#### 코드 예시 및 분석

```cpp
glm::vec4 vec(1.0f, 0.0f, 0.0f, 1.0f);
glm::mat4 trans = glm::mat4(1.0f);
trans = glm::translate(trans, glm::vec3(1.0f, 1.0f, 0.0f));
vec = trans * vec;

std::cout << vec.x << vec.y << vec.z << std::endl;
```

*   **`glm::vec4 vec(...)`**: GLM 내장 클래스를 사용하여 (1,0,0) 위치 벡터를 정의한다. 3D 공간의 이동 연산을 동차 좌표계 행렬곱으로 풀기 위해 $w$요소(1.0f)를 추가한 `vec4` (1,0,0,1) 형태로 선언한다. **GLM의 벡터는 기본적으로 열 벡터(Column Vector)** 로 연산된다.
*   **`glm::mat4 trans = glm::mat4(1.0f);`**: 주대각선 값이 모두 1인 순수한 4x4 항등 행렬(Identity Matrix)을 주조한다. 초기 변환이 없는 상태에서 시작해야 한다.
*   **`glm::translate(...)`**: 항등 행렬 `trans`에 $(1.0, 1.0, 0.0)$ 만큼의 위치 변화량을 주입해 이동 변환 행렬을 빚어낸다. 
*   **`vec = trans * vec;`**: 최종 완성된 이동 변환 행렬 `trans`와 원본 위치 벡터 `vec`를 곱한다. 이 수학적 곱셈 연산 단 한 번으로 점이 새로운 위치로 순간 이동한다.

### 2. 회전 (Rotation) 및 스케일링 (Scale) 

이동 메커니즘을 응용하여 객체를 특정 각도만큼 회전시키거나 전체 크기를 조절하는(Scale) 복합 변환도 똑같은 방식으로 행렬을 곱하여 수행한다.

#### 코드 예시 

```cpp
glm::mat4 transform = glm::mat4(1.0f);
// Z축 중심(0, 0, 1)으로 90도(라디안 변환) 회전하는 행렬 추가
transform = glm::rotate(transform, glm::radians(90.0f), glm::vec3(0.0, 0.0, 1.0));
// 기존 크기에서 X, Y, Z축 방향으로 절반(0.5)만큼 축소시키는 행렬 추가
transform = glm::scale(transform, glm::vec3(0.5, 0.5, 0.5));
```

![Rotation and Scale Matrix Concept](/images/GLM2.jpg){: width="500" }

> [!important]
> **행렬 곱셈의 순서 주의**  
> 행렬 연산은 교환법칙이 성립하지 않는다. 코드 작성 시 크기 조절(Scale), 회전(Rotate), 이동(Translate) 순서로 적용해야 의도한 물리학적 변환 결과가 도출된다. 만약 이동을 먼저 하고 회전을 나중에 하면 객체가 화면 밖으로 크게 공전해버리게 된다.

## 실제 렌더링 파이프라인에 적용하기

행렬 곱셈은 CPU 쪽인 애플리케이션(C++) 코드에서 수행하는 것보다, 막강한 부동소수점 병렬 연산 유닛이 몰려있는 GPU 환경의 **버텍스 셰이더**에 넘겨서 정점마다 처리하도록 두는 것이 압도적으로 효율적이다.

### 3. 버텍스 셰이더 (Vertex Shader) 코드 작성

앞서 C++에서 생성한 이 4x4 변환 행렬 `transform`을 유니폼 변수로 선언하고 GPU로 전달받아 위치 벡터에 적용해야 한다.

```glsl
#version 330 core
layout (location = 0) in vec3 aPos;
layout (location = 1) in vec2 aTexCoord;

out vec2 TexCoord;

// 외부 C++ 프로그램으로부터 전달받을 4x4 변환 행렬
uniform mat4 transform;

void main()
{
    // 정점의 로컬 좌표(aPos)에 변환 행렬을 곱하여 최종 렌더링 좌표계를 산출한다.
    gl_Position = transform * vec4(aPos, 1.0f);
    TexCoord = vec2(aTexCoord.x, aTexCoord.y);
}
```
{: file="shader.vert" }

`gl_Position` 내장 변수를 산출할 때 마지막에 `transform` 행렬을 한 번 앞단에서 곱해줌으로써 수백~수만 개의 정점(Vertex) 데이터들이 일제히 이동/회전/분류되는 기하학적 파이프라인 마법이 성립된다.

### 4. OpenGL 코드 측 행렬 전송

준비가 끝난 버텍스 셰이더를 향해 C++ 측에서 변환 행렬 공간(메모리)을 검색하고 주입하는 과정이다.

```cpp
// 셰이더 프로그램 내부의 'transform' 이름의 유니폼 레지스터 주소 로드
unsigned int transformLoc = glGetUniformLocation(ourShader.ID, "transform");

// 로컬에서 계산한 GLM 변환 행렬의 값을 해당 주소로 포인터를 통해 캐스팅 전송
glUniformMatrix4fv(transformLoc, 1, GL_FALSE, glm::value_ptr(transform));
```
{: file="main.cpp" }

1.  **`transformLoc`**: `glGetUniformLocation()` 함수를 통해 대상 셰이더 내부의 `transform` 유니폼 변수 메모리 위치(인덱스)를 탐색한다.
2.  **`1`**: 넘겨줄 행렬의 개수. 여기서는 하나의 단일 4x4 행렬만 넘기므로 1이다. 만약 조인트 애니메이션 등 수십 개의 본 매트릭스를 던진다면 배열 개수가 된다.
3.  **`GL_FALSE`**: Transpose(행렬 대각 전치 처리) 여부 옵션이다. GLM은 OpenGL이 선호하는 Column-Major(열 우선) 정렬 구조를 그대로 따르므로 전치할 필요가 없어 `GL_FALSE`를 기입한다.
4.  **`glm::value_ptr(...)`**: **GLM은 그레이트한 수학 라이브러리지만 내부 클래스 객체 형태로 감싸져 있다**. 따라서 순수한 C 포인터 형식의 메모리를 기대하는 OpenGL 내장 함수 형식을 맞춰주기 위해 `glm::value_ptr()` 래핑 함수를 사용해 메모리 시작 주소를 강제 변환하여 전달한다.

### 5. 시간에 따른 리얼타임 회전 효과 

시간의 흐름 값을 회전 각도값에 지속적으로 투입하면, 정지된 정적 변환이 아닌 역동적으로 공전/자전하는 애니메이션 효과를 구현할 수 있다. 렌더링 루프(While) 내부에서 코드를 갱신시켜주면 된다.

```cpp
glm::mat4 transform = glm::mat4(1.0f);
transform = glm::translate(transform, glm::vec3(0.5f, -0.5f, 0.0f));
// 시간의 흐름(glfwGetTime)을 회전의 라디안 각도로 바로 꽂아 넣는다
transform = glm::rotate(transform, (float)glfwGetTime(), glm::vec3(0.0f, 0.0f, 1.0f));
```

`glm::rotate` 함수의 두 번째 인수는 회전 라디안 값(Angle)을 요구한다. `glfwGetTime()`은 프로그램 시작 시점으로부터 흐른 초 단위 시간을 실수 폼으로 반환하므로, 이를 그대로 얹어주면 Z축(`0.0, 0.0, 1.0`)을 기준으로 초당 일정한 속도로 물체가 계속해서 팽이처럼 빙글빙글 회전하는 애니메이션 루프가 완성된다.
