## GLM 라이브러리를 활용한 수학적 변환 구현

**GLM (OpenGL Mathematics)**은 OpenGL 환경에서 벡터와 행렬 연산을 처리할 수 있도록 설계된 C++ 수학 라이브러리입니다.

본 포스트에서는 특정 점 $(1,0,0)$을 $(1,1,0)$ 위치로 **이동(Translation)**시키고 **회전(Rotation)**시키는 예제를 통해 GLM의 핵심 API 활용법을 기술합니다.

### 1. 이동 (Translation) 

이동(Translation)은 공간 상에서 객체의 위치 좌표를 옮기는 선형 대수학적 변환이다. GLM을 사용하면 최소한의 코드로 4x4 이동 변환 행렬을 생성하고 특정 벡터 위치에 적용할 수 있다.

수학적으로 이동 변환은 항등 행렬에 이동값 벡터를 조합하여 구성하며, 벡터와의 행렬 곱 형태로 계산된다.
$$
\begin{bmatrix}
1 & 0 & 0 & T_x \\
0 & 1 & 0 & T_y \\
0 & 0 & 1 & T_z \\
0 & 0 & 0 & 1
\end{bmatrix}
\cdot
\begin{bmatrix}
x \\ y \\ z \\ 1
\end{bmatrix}
=
\begin{bmatrix}
x + T_x \\
y + T_y \\
z + T_z \\
1
\end{bmatrix}
$$

#### 코드 예시 및 분석

```cpp
glm::vec4 vec(1.0f, 0.0f, 0.0f, 1.0f);
glm::mat4 trans = glm::mat4(1.0f);
trans = glm::translate(trans, glm::vec3(1.0f, 1.0f, 0.0f));
vec = trans * vec;

std::cout << vec.x << vec.y << vec.z << std::endl;
```

*   **`glm::vec4 vec(...)`**: GLM 클래스를 사용하여 위치 벡터를 정의합니다. 4x4 행렬 연산을 위해 $w$ 요소(1.0f)를 포함하는 `vec4` 형식을 사용하며, **GLM의 벡터는 기본적으로 열 벡터(Column Vector)**로 처리됩니다.
*   **`glm::mat4 trans = glm::mat4(1.0f);`**: 4x4 항등 행렬(Identity Matrix)을 생성합니다.
*   **`glm::translate(...)`**: 항등 행렬에 위치 변화량을 적용하여 이동 변환 행렬을 생성합니다.
*   **`vec = trans * vec;`**: 이동 변환 행렬과 원본 위치 벡터를 곱하여 결과 좌표를 계산합니다.

### 2. 회전 (Rotation) 및 스케일링 (Scale) 

객체를 특정 각도만큼 회전시키거나 크기를 조절하는 복합 변환도 동일한 방식으로 행렬 연산을 통해 수행한다.

#### 코드 예시 

```cpp
glm::mat4 transform = glm::mat4(1.0f);
// Z축(0, 0, 1)을 기준으로 90도 회전하는 행렬 적용
transform = glm::rotate(transform, glm::radians(90.0f), glm::vec3(0.0, 0.0, 1.0));
// X, Y, Z축 방향으로 0.5배 축소하는 행렬 적용
transform = glm::scale(transform, glm::vec3(0.5, 0.5, 0.5));
```

![Rotation and Scale Matrix Concept](/images/GLM2.jpg){: width="500" }

> [!important]
> **행렬 연산 순서**  
> 행렬 곱셈은 교환법칙이 성립하지 않으므로 적용 순서에 주의해야 한다. 일반적으로 크기 조절(Scale), 회전(Rotate), 이동(Translate) 순서로 적용해야 의도한 변환 결과가 산출된다.

## 렌더링 파이프라인 적용

행렬 연산은 CPU보다 GPU의 **버텍스 셰이더**에서 처리하는 것이 효율적이다.

### 3. 버텍스 셰이더 (Vertex Shader)

애플리케이션에서 생성한 4x4 변환 행렬 `transform`을 유니폼 변수로 선언하고 정점 위치 벡터에 적용한다.

```glsl
#version 330 core
layout (location = 0) in vec3 aPos;
layout (location = 1) in vec2 aTexCoord;

out vec2 TexCoord;

// 4x4 변환 행렬 유니폼
uniform mat4 transform;

void main()
{
    // 정점 좌표(aPos)에 변환 행렬을 곱하여 최종 좌표를 산출한다.
    gl_Position = transform * vec4(aPos, 1.0f);
    TexCoord = vec2(aTexCoord.x, aTexCoord.y);
}
```

### 4. 행렬 데이터 전송

C++ 측에서 변환 행렬의 위치를 탐색하고 데이터를 전송하는 과정이다.

```cpp
// 유니폼 변수 위치 탐색
unsigned int transformLoc = glGetUniformLocation(ourShader.ID, "transform");

// GLM 변환 행렬 데이터를 전송
glUniformMatrix4fv(transformLoc, 1, GL_FALSE, glm::value_ptr(transform));
```
{: file="main.cpp" }

1.  **`glGetUniformLocation()`**: 셰이더 내부의 `transform` 유니폼 변수 메모리 위치를 탐색한다.
2.  **`1`**: 전송할 행렬의 개수.
3.  **`GL_FALSE`**: 전치(Transpose) 여부. GLM은 OpenGL과 동일한 열 우선(Column-Major) 구조를 사용하므로 `GL_FALSE`를 지정한다.
4.  **`glm::value_ptr(...)`**: GLM 객체 내부의 데이터 포인터를 획득하여 전달한다.

### 5. 실시간 회전 애니메이션 구현 

렌더링 루프 내에서 시간 값을 회전 각도에 반영하여 동적인 효과를 구현할 수 있다.

```cpp
glm::mat4 transform = glm::mat4(1.0f);
transform = glm::translate(transform, glm::vec3(0.5f, -0.5f, 0.0f));
// 시간 흐름(glfwGetTime)을 회전 각도로 입력
transform = glm::rotate(transform, (float)glfwGetTime(), glm::vec3(0.0f, 0.0f, 1.0f));
```

`glm::rotate` 함수의 각도 인자에 `glfwGetTime()` 값을 활용하면 Z축을 기준으로 지속적으로 회전하는 애니메이션이 수행된다.
