## 1. 카메라 직접 움직이기 (키보드 조작)

루프를 도는 자동 카메라 워킹을 넘어, 직접 키보드 WASD 키를 눌러 1인칭 시점으로 씬을 탐험하는 컨트롤 시스템을 구축해 보았다. 위치 연산을 위해 카메라의 기준점 변수 3가지를 전역 공간에 할당한다.

```cpp
glm::vec3 cameraPos   = glm::vec3(0.0f, 0.0f,  3.0f);
glm::vec3 cameraFront = glm::vec3(0.0f, 0.0f, -1.0f);  // 카메라가 응시하는 정면 방향 벡터
glm::vec3 cameraUp    = glm::vec3(0.0f, 1.0f,  0.0f);
```
{: file="main.cpp" }

뷰 행렬을 생성하는 `lookAt` 함수는 이제 렌더 루프 매 프레임마다 변동되는 이 동적 변수들을 참조하도록 세팅한다.

```cpp
// 타겟 위치 = 현재 발조심(Position) + 쳐다보는 시선 방향(Front 방향 벡터)
view = glm::lookAt(cameraPos, cameraPos + cameraFront, cameraUp);
```
{: file="main.cpp" }

이전에 창 종료(ESC) 감지를 위해 짰던 `processInput` 함수를 재활용하여 WASD 키보드 입력 핸들러 로직을 심어준다.

```cpp
void processInput(GLFWwindow *window)
{
    // ... ESC 처리 로직 ...
    const float cameraSpeed = 0.05f; 
    
    // W, S 키: 전진 및 후진. 현재 위치에서 전방(Front) 벡터를 더하거나 뺀다.
    if (glfwGetKey(window, GLFW_KEY_W) == GLFW_PRESS)
        cameraPos += cameraSpeed * cameraFront;
    if (glfwGetKey(window, GLFW_KEY_S) == GLFW_PRESS)
        cameraPos -= cameraSpeed * cameraFront;
        
    // A, D 키: 좌우 게걸음(Strafe). 전방(Front)과 위쪽(Up) 벡터를 외적하여 가로축(Right) 방향을 구한 뒤 그 방향으로 이동시킨다.
    if (glfwGetKey(window, GLFW_KEY_A) == GLFW_PRESS)
        cameraPos -= glm::normalize(glm::cross(cameraFront, cameraUp)) * cameraSpeed;
    if (glfwGetKey(window, GLFW_KEY_D) == GLFW_PRESS)
        cameraPos += glm::normalize(glm::cross(cameraFront, cameraUp)) * cameraSpeed;
}
```
{: file="main.cpp" }

> [!important]
> 좌우 이동 키(A, D) 구현부에서 교차 외적(Cross) 산출 후 무조건 **`glm::normalize` 로 정규화**를 수행하여 벡터 길이를 1로 맞춘 점에 주목해야 한다. 이를 누락하면 대각선 이동 시 전진/측면 벡터 길이가 무분별하게 합산되어 캐릭터가 비정상적으로 튀어 나가는(Run Boost) 버그가 발생한다.

## 2. 프레임 동기화 이동 (Movement Speed & DeltaTime)

수치 변수 `cameraSpeed` 를 루프 1바퀴당 0.05 이동으로 하드코딩 하면 결함이 발생한다.
구형 사무용 PC(초당 30루프)에서는 캐릭터가 느리게 움직이고, 하이엔드 게이밍 PC(초당 300루프)에서는 순식간에 맵 밖으로 이동하게 된다. 컴퓨터 성능에 관계없이 **"1초당 동일한 거리"를 안정적으로 이동하게 하려면, 프레임 처리 지연 시간인 `deltaTime` 을 이동 계수로 곱해주어야 한다**.

`deltaTime` 이란 직전 프레임을 렌더링하고 모니터 스크린에 출력하기까지 걸린 실제 시간 간격(초)을 의미한다.

```cpp
// 프레임 딜레이 타이머를 전역에 선언
float deltaTime = 0.0f;  
float lastFrame = 0.0f;  
```
{: file="main.cpp" }

루프의 상단에서 매 프레임 시간 격차를 최신화한다.
```cpp
float currentFrame = glfwGetTime();
deltaTime = currentFrame - lastFrame;
lastFrame = currentFrame;
```
{: file="main.cpp" }

이제 하드웨어 성능에 의존하지 않는 규격화된 이동 속도를 `processInput` 내부에서 산출한다.

```cpp
void processInput(GLFWwindow *window)
{
    // 속도 상수 2.5 단위를 세팅하고 딜레이 타임을 팩터로 묶어 곱셈
    float cameraSpeed = 2.5f * deltaTime;
    // ... [하단 프론트/사이드 이동 로직은 동일]
}
```
{: file="main.cpp" }

## 3. Look Around (마우스 시점 뷰 제어)

걸어 다니는 법을 배웠으니, 마우스 커서를 움직여 시점을 전환하는 1인칭 프리캠 시스템을 연동할 차례다.

![Pitch Yaw Roll](https://learnopengl.com/img/getting-started/camera_smooth.png){: width="500" }  
_비행 시뮬레이터에서 유래된 Euler Angles 축_

### 3.1. 오일러 각도 (Euler Angle) 
오일러 각도는 3D 공간의 회전을 3축으로 나누어 정의한다. 바로 **Pitch(상하 회전), Yaw(좌우 회전), Roll(기울기)** 이다. 
이 3개의 축 각도를 설정하고 융합하면 모든 방향의 시점 회전 벡터를 계산할 수 있다. 일반적인 FPS 카메라에서는 멀미를 방지하기 위해 Roll 회전을 배제하고 `pitch` 와 `yaw` 값만 추적하여 관리한다.

### 3.2. 삼각함수로 방향 벡터 유추하기

![Triangle](https://learnopengl.com/img/getting-started/camera_triangle.png){: width="400" }  

밑변이 1로 고정된 직각삼각형에서의 삼각함수 공식은 매우 직관적이다.
* `인접변 (밑변) = cos(각도)`
* `대향변 (높이) = sin(각도)`

이 공식을 마우스 시점 좌표계에 적용하여, Z/X 평면 상에서 좌우 **Yaw 각도** 회전 시 축의 길이가 어떻게 변하는지 분석해 본다.

![Yaw Vector](https://learnopengl.com/img/getting-started/camera_yaw.png){: width="400" }    

카메라의 렌즈 방향을 기준으로 삼각함수를 대입한다면 시점 방향 벡터의 파편을 다음과 같이 조립할 수 있다.

```cpp
glm::vec3 direction;
// C++의 sin, cos 함수 입력 인자는 반드시 '라디안(Radian)' 체계여야 한다.
direction.x = cos(glm::radians(yaw)); 
direction.z = sin(glm::radians(yaw));
```
{: file="main.cpp" }

X축과 Z축 투영 길이를 도출했으므로 상하를 보는 **Pitch 각도**까지 구면 변환 방정식에 결합시켜 입체 좌표를 계산한다.

```cpp
// 3축 오일러 시야각 최종 계산식
direction.x = cos(glm::radians(yaw)) * cos(glm::radians(pitch));
direction.y = sin(glm::radians(pitch)); // 상하 Y축은 오직 Pitch가 담당한다.
direction.z = sin(glm::radians(yaw)) * cos(glm::radians(pitch));
```
{: file="main.cpp" }

## 4. 카메라 초기 시점 방향 불일치(Mismatch) 이슈

위의 삼각함수 수학 공식에는 숨겨진 함정이 존재한다. 

초기 환경에서 `yaw = 0`, `pitch = 0` 인 정자세 상태로 앱이 실행된다고 가정해 보면 다음과 같은 결과가 나온다.

### 4.1. Yaw = 0° 수식 도출 과정
```text
direction.x = cos(0) * cos(0) => 1 * 1 = 1
direction.y = sin(0)          => 0
direction.z = sin(0) * cos(0) => 0 * 1 = 0
```
최종적으로 산출되는 Direction 전방 벡터는 `(1, 0, 0)` 이 된다. 
이는 플레이어 카메라가 모니터 정면(Z축 음의 방향)을 쳐다보지 않고, **화면의 우측 90도 방향(X축 양의 방향)을 쳐다보며 시작한다**는 의미다. 씬 내부의 오브젝트들은 주로 Z축 음의 방향에 배치되어 있으므로 문제가 된다.

### 4.2. 해결책: 초기 Yaw 각도 오프셋 보정

카메라 렌즈가 기본적으로 사용자가 인지하는 모니터 정면(음의 Z축 방향)을 향하도록 설정하려면 보정이 필요하다. 
현재 `(1, 0, 0)` X축 양의 방향을 향하고 있는 시선을 정면 Z축 방향으로 **시계 방향 90도 회전**시키면 해결된다.

OpenGL 수학 체계에서는 **양수 각도가 반시계(CCW) 회전을 의미**하므로, 시계 방향으로 회전시키려면 초기 `yaw` 변수 값을 0이 아닌 **`-90.0f`** 로 설정해야 한다.

```cpp
// 전역 마우스 좌표계의 초기 편향 각도 부여
float yaw = -90.0f; 
```
{: file="main.cpp" }

이렇게 `-90`도 초기 값을 부여한 채 계산식을 적용하면 도출되는 벡터가 `(0, 0, -1)` 이 되며, 완벽하게 음의 Z축 방향을 응시하게 되어 문제가 해결된다.
