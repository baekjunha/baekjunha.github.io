## 1. 마우스 입력 처리 (시점 전환)

**자료 출처**: [LearnOpenGL](https://learnopengl.com/)

3D 공간의 회전축인 `Yaw(좌우 회전)`와 `Pitch(상하 회전)` 값은 마우스의 움직임을 통해 산출한다. 수평 이동은 Yaw에, 수직 이동은 Pitch에 영향을 준다. 핵심 구현 원리는 **이전 프레임의 마우스 좌표를 저장하고, 현재 프레임과의 변위(Offset)를 계산하여 Pitch/Yaw 값을 갱신하는 것**이다.

### 1.1. 마우스 커서 설정 (FPS 모드)

FPS 환경에서는 커서가 화면 영역을 벗어나 조작 오류를 유발하는 것을 방지해야 한다. 이를 위해 GLFW의 입력 모드를 설정하여 커서를 숨기고 윈도우 내부에 캡처한다.

*   **비활성화**: 화면상에서 마우스 커서를 은닉한다.
*   **캡처**: 응용 프로그램 창이 포커스를 유지하는 동안 커서 위치를 창 중앙으로 고정한다.

```cpp
glfwSetInputMode(window, GLFW_CURSOR, GLFW_CURSOR_DISABLED);
```
{: file="main.cpp" }

### 1.2. 마우스 콜백(Callback) 등록

마우스 이동 이벤트를 처리하기 위해 GLFW의 표준 프로토타입에 따른 콜백 함수를 정의한다.

```cpp
void mouse_callback(GLFWwindow* window, double xpos, double ypos);
```
{: file="main.cpp" }

정의된 함수를 GLFW 이벤트 루프에 등록한다.
```cpp
glfwSetCursorPosCallback(window, mouse_callback);
```
{: file="main.cpp" }

### 1.3. 방향 벡터(Direction Vector) 산출 프로세스

**1. 마우스 변위(Offset) 계산**
초기 마우스 위치를 설정한 후, 콜백 함수 내에서 직전 위치와의 차이를 계산한다.

```cpp
float lastX = 400, lastY = 300;
```
{: file="main.cpp" }

```cpp
float xoffset = xpos - lastX;
float yoffset = lastY - ypos; // Y축은 스크린 좌표계 특성에 따라 역순 계산
lastX = xpos;
lastY = ypos;

const float sensitivity = 0.1f; // 감도 계수 적용
xoffset *= sensitivity;
yoffset *= sensitivity;
```
{: file="main.cpp" }

**2. 오일러 각(Yaw, Pitch) 갱신**
```cpp
yaw   += xoffset;
pitch += yoffset;
```
{: file="main.cpp" }

**3. 각도 제한 (Gimbal Lock 방지)**
시점 역전 현상을 방지하기 위해 Pitch 각도를 `89.0f`와 `-89.0f` 사이로 제한한다.

```cpp
if (pitch > 89.0f)
    pitch = 89.0f;
if (pitch < -89.0f)
    pitch = -89.0f;
```
{: file="main.cpp" }

**4. 최종 방향 벡터 계산**
삼각함수를 사용하여 최종 전방(Front) 벡터를 산출하고 정규화한다.
```cpp
glm::vec3 direction;
direction.x = cos(glm::radians(yaw)) * cos(glm::radians(pitch));
direction.y = sin(glm::radians(pitch));
direction.z = sin(glm::radians(yaw)) * cos(glm::radians(pitch));

cameraFront = glm::normalize(direction);
```
{: file="main.cpp" }

## 2. 초기 입력 예외 처리

프로그램 구동 직후 캡처된 커서 좌표와 초기 설정값 사이의 큰 편차로 인해 시점이 급격히 도약하는 현상이 발생할 수 있다. 이를 방지하기 위해 최초 입력 여부를 확인하는 상태 변수를 도입하여 좌표를 동기화한다.

```cpp
bool firstMouse = true;

if (firstMouse) 
{
    lastX = xpos;
    lastY = ypos;
    firstMouse = false;
}
```

## 3. 마우스 휠을 이용한 줌(Zoom) 기능

시야각(Field of View, FOV) 조정을 통해 확대/축소 효과를 구현한다. FOV 각도를 좁히면 화면이 확대되고, 넓히면 줌 아웃 효과가 발생한다.

```cpp
float fov = 45.0f;

void scroll_callback(GLFWwindow* window, double xoffset, double yoffset)
{
    fov -= (float)yoffset;
    if (fov < 1.0f)
        fov = 1.0f;
    if (fov > 45.0f)
        fov = 45.0f;
}
```
{: file="main.cpp" }

갱신된 `fov` 변수는 매 프레임 투영 행렬(`Projection Matrix`) 생성 시 반영되어 셰이더의 유니폼 변수로 전송되어야 한다.

```cpp
// 렌더링 루프 내부
glm::mat4 projection = glm::perspective(glm::radians(fov), 800.0f / 600.0f, 0.1f, 100.0f);
```
{: file="main.cpp" }

```cpp
glfwSetScrollCallback(window, scroll_callback);
```
{: file="main.cpp" }

## 4. Camera 클래스 모듈화

카메라 로직이 복잡해짐에 따라 이를 별도의 클래스로 캡슐화하여 관리하는 것이 유지보수 측면에서 유리하다. 추상화된 `Camera Class`를 통해 다양한 시점 전환 및 제어를 효율적으로 수행할 수 있다.

> [!info] 
> 오일러 각 기반 카메라는 일반적인 3D 환경에 적합하나, 수직 회전이 잦은 비행 시뮬레이션 등에서는 짐벌 락(Gimbal Lock) 문제를 해결하기 위해 쿼터니언(Quaternion) 수학 기반의 구현이 권장된다.
