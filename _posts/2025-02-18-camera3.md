## 1. 마우스 입력 처리 (프리캠 시점 전환)

**자료 출처**: [LearnOpenGL](https://learnopengl.com/)

3D 공간의 머리 회전 축인 `Yaw(좌우 회전)`와 `Pitch(상하 회전)` 값은 보통 마우스나 조이스틱의 움직임에서 얻어온다. 수평 마우스 이동은 Yaw에 영향을 주고, 수직 마우스 이동은 Pitch에 영향을 준다. 
핵심 아이디어는 **이전 프레임의 마우스 터치 위치를 저장한 뒤, 현재 프레임에서 마우스가 XY축으로 얼마나 이동했는지 델타(변위 오프셋)를 계산하는 것**이다. 변위가 클수록 Pitch/Yaw 값을 더 크게 누적하여 카메라의 시선을 획기적으로 돌린다.

자동차 핸들을 조금 돌리면 바퀴가 조금 꺾이고, 휙 돌리면 급커브를 도는 것과 동일한 직관적인 원리다.

### 1.1. 마우스 커서 숨기기 및 화면 중앙 캡처 (FPS 모드)

FPS 게임을 할 때, 윈도우 OS의 흰색 마우스 커서가 화면 중앙에 둥둥 떠 있는 것을 본 적은 없을 것이다. 커서가 노출되면 조준 몰입도가 깨질 뿐만 아니라, 마우스를 거칠게 튕기다가 창 밖으로 커서가 탈주해 바탕화면을 광클하는 끔찍한 조작 미스가 발생한다.

이를 방지하기 위해 GLFW에게 **커서를 숨기고 윈도우 창 중앙에 가두도록(캡처)** 지시해야 한다.
*   **숨기기**: 렌더링 화면 상에서 마우스 포인터 그래픽을 투명하게 지운다.
*   **캡처하기**: 응용 프로그램 창이 포커스를 쥐고 있는 한, 마우스를 아무리 밖으로 움직여도 실제 시스템상의 커서 위치는 강제로 창 한가운데 중앙 좌표에 영구 고정된다.

단 한 줄의 강력한 C++ 호출로 이 두 가지 모드를 동시에 발동시킬 수 있다.

```cpp
glfwSetInputMode(window, GLFW_CURSOR, GLFW_CURSOR_DISABLED);
```
{: file="main.cpp" }

이 세팅이 먹히면 마우스는 보이지도 않고 창 밖으로 탈주하지도 않는다. 완벽한 1인칭 FPS 카메라 기반 조작 환경이 완성된 것이다.

### 1.2. 마우스 콜백(Callback) 연동

마우스가 패드 위를 1픽셀이라도 미끄러질 때마다, 그 움직임 이벤트를 실시간으로 가로채 처리할 리스너(Listener) 함수를 하나 파야 한다. GLFW의 표준 프로토타입에 맞춰 콜백 함수 껍데기를 만든다.

```cpp
void mouse_callback(GLFWwindow* window, double xpos, double ypos);
```
{: file="main.cpp" }

여기서 `xpos`와 `ypos`는 창 기준 최신 마우스 위치 좌표를 의미한다. 이 함수를 GLFW 메인 이벤트 루프에 등록해준다.
```cpp
glfwSetCursorPosCallback(window, mouse_callback);
```
{: file="main.cpp" }

### 1.3. 방향 벡터(Direction Vector) 도출 4단계 알고리즘

콜백 안에서 마우스 좌표값을 넘겨받아 최종 카메라 렌즈 방향인 3D 벡터를 뽑아내려면 다음 4가지 단계를 철저히 밟아야 한다.

**1. 직전 프레임 대비 마우스 변위(Offset) 계산**
최초 1회에는 마우스가 화면 정중앙(800x600 해상도 기준 400, 300)에 있다고 초기값을 설정해 둔다.

```cpp
float lastX = 400, lastY = 300;
```
{: file="main.cpp" }

콜백 함수 내에서 직전 위치 `last` 값과 방금 들어온 `pos` 값의 차이를 뺀다.
```cpp
// X축은 우측이 양수라 그대로 뺀다.
float xoffset = xpos - lastX;
// Y축은 밑으로 갈수록 커지는 스크린 좌표계이므로 수학 좌표계와 반대로 뺀다.
float yoffset = lastY - ypos; 
lastX = xpos;
lastY = ypos;

// 너무 휙휙 돌아가지 않도록 감도(Sensitivity)를 곱해 속력을 죽여준다.
const float sensitivity = 0.1f;
xoffset *= sensitivity;
yoffset *= sensitivity;
```
{: file="main.cpp" }

**2. 변위를 전역 오일러 각(Yaw, Pitch)에 누적 합산**
```cpp
yaw   += xoffset;
pitch += yoffset;
```
{: file="main.cpp" }

**3. 화면 한계 각도 락(Lock) 걸기 (짐벌 락 방지)**
사용자가 고개를 위로 90도 넘게 과하게 젖히면 세상이 거꾸로 뒤집히는 이른바 **LookAt 함수의 역전 현상(Gimbal Lock의 아종)** 이 일어난다. 기괴한 엑소시스트 고개 꺾기를 방지하기 위해, 최대/최소 Pitch 각도를 `89.0f` 와 `-89.0f` 로 리미트를 건다. (90도로 딱 맞추면 평행 에러가 터진다).

```cpp
if (pitch > 89.0f)
    pitch = 89.0f;
if (pitch < -89.0f)
    pitch = -89.0f;
```
{: file="main.cpp" }
수평 도리도리(Yaw)는 360도 뱅뱅 돌아도 화면이 뒤집힐 일이 없으므로 락을 걸지 않는다.

**4. 3축 오일러 벡터 계산 및 반영**
이전 챕터에서 배운 삼각함수 로직으로 최종 Front 벡터를 조립해 정규화(Normalize) 해준다.
```cpp
glm::vec3 direction;
direction.x = cos(glm::radians(yaw)) * cos(glm::radians(pitch));
direction.y = sin(glm::radians(pitch));
direction.z = sin(glm::radians(yaw)) * cos(glm::radians(pitch));

cameraFront = glm::normalize(direction);
```
{: file="main.cpp" }

## 2. '퍼스트 클릭' 순간 이동 버그 막기

프로그램이 탁 켜지고 커서 창이 최초 캡처 포커싱을 받는 찰나에 어마어마한 화면 순간이동(Jump) 버그가 발생한다. 원인은 자명하다. 구석에 쳐박혀 있던 윈도우 커서가 앱 포커싱을 먹으며 강제로 좌표(가령 `xpos=10, ypos=10`) 에 끌려갈 때, 우리의 `lastX=400` 중앙 초기값과 엄청난 편차가 발생하여 `xoffset` 변위가 390씩 터지는 대형 사고가 나는 것이다. 

이 오버플로우를 막으려면 앱 구동 후 마우스 입력이 **생전 처음 들어왔는지** 체크하는 가망 상태 변수(Flag)가 필요하다. 처음 들어왔을 때만 현재 커서 좌표를 Base Location으로 무조건 동기화(Sync) 시켜 오프셋을 0으로 융해시킨다.

```cpp
bool firstMouse = true;

// 콜백 함수 내부
if (firstMouse) 
{
    lastX = xpos;
    lastY = ypos;
    firstMouse = false;
}
```
{: file="main.cpp" }

### 2.1. 완성된 마우스 시점 제어 콜백 함수

모든 예외처리가 끝난 `mouse_callback` 최종 로직은 다음과 같다.

```cpp
void mouse_callback(GLFWwindow* window, double xpos, double ypos)
{
    if (firstMouse) // 최상단에 퍼스트 터치 방어막
    {
        lastX = xpos;
        lastY = ypos;
        firstMouse = false; 
    }
  
    float xoffset = xpos - lastX;
    float yoffset = lastY - ypos; 
    lastX = xpos;
    lastY = ypos;

    float sensitivity = 0.1f;
    xoffset *= sensitivity;
    yoffset *= sensitivity;

    yaw   += xoffset;
    pitch += yoffset;

    if (pitch > 89.0f)
        pitch = 89.0f;
    if (pitch < -89.0f)
        pitch = -89.0f;

    glm::vec3 direction;
    direction.x = cos(glm::radians(yaw)) * cos(glm::radians(pitch));
    direction.y = sin(glm::radians(pitch));
    direction.z = sin(glm::radians(yaw)) * cos(glm::radians(pitch));
    cameraFront = glm::normalize(direction);
}
```
{: file="main.cpp" }

## 3. 마우스 휠로 카메라 줌(Zoom-in/out) 조작

FPS 게임의 저격 소총 무배율 스코프 줌처럼, 마우스 스크롤 휠을 드르륵 굴려 화면 확대율을 건드리는 상호작용을 파볼 차례다.

시야각(Field of View, FOV) 변수는 씬을 렌즈에 담는 화각의 넓이를 관장한다. 광각 렌즈처럼 FOV 값 각도를 넓히면 투영되는 월드의 부피가 스크린 구석구석 더 많이 담겨 상대적으로 줌 아웃 축소되어 보이고, 각도를 좁히면 화면 한가운데 특정 영역만 풀사이즈 모니터에 영사되어 기하급수적으로 확대(Zoom-in) 되는 원리다.

스크롤 신호를 받는 리스너 콜백 껍데기를 파보자.
```cpp
float fov = 45.0f;

void scroll_callback(GLFWwindow* window, double xoffset, double yoffset)
{
    // 스크롤 방향 부호에 따라 FOV 각도 증감
    fov -= (float)yoffset;
    
    // 시야각 1도 미만(극단적 줌인 파괴 방지) 혹은 기본 광각 45도 초과 락 걸기
    if (fov < 1.0f)
        fov = 1.0f;
    if (fov > 45.0f)
        fov = 45.0f;
}
```
{: file="main.cpp" }

`yoffset` 값이 위로 굴릴 때와 아래로 굴릴 때 `+1, -1` 틱 신호를 던져주므로, 이를 FOV 각도에 가감산한다.

이제 뷰포트 원근을 왜곡하는 투영 행렬(`Projection Matrix`)에 이 가변식 `fov` 변수를 실시간 갱신 삽입한다.

> [!tip]
> `fov` 변수는 사용자가 휠을 굴릴 때마다 런타임에 동적으로 변경된다. 따라서 투영 행렬 갱신 코드는 단 한 번만 실행되는 초기화 블록이 아니라, 초당 수십 번 도는 `while` 메인 루프 안에 세팅하여 프레임 렌더링 때마다 셰이더의 uniform 변수로 퍼올려(Upload) 줘야만 부드러운 화각 변화가 화면에 작화된다.

```cpp
// 렌더링 루프 내부
glm::mat4 projection = glm::perspective(glm::radians(fov), 800.0f / 600.0f, 0.1f, 100.0f);
```
{: file="main.cpp" }

마지막으로 이 아름다운 스크롤 콜백 리스너를 OS 시스템에 정식으로 등록한다.
```cpp
glfwSetScrollCallback(window, scroll_callback);
```
{: file="main.cpp" }

## 4. 자체 Camera 클래스 객체화 도입

지금까지 카메라 로직을 `main.cpp` 최상단 전역 변수로 덕지덕지 발라 놨지만, 씬이 더 복잡해지고 여러 카메라 뷰(예: 미니맵 직교 투영 카메라, 플레이어 1인칭 카메라)를 스위칭해야 한다면 코드는 곧 통제 불능의 스파게티가 될 것이다.

앞으로 다가올 상위 챕터 실습에서는 이 반복적인 행렬 수학 식을 하나하나 나열하지 않고, 셰이더 클래스를 만들었듯 시야 조작 코어들을 몽땅 숨겨 추상화된 단일 헤더 `Camera Class` 파일로 모듈화할 것이다. 

> [!info] 
> 우리가 만든 이 범용(Fly Style) 카메라는 보통의 오일러 기반 3D 작업에 유용하지만, 90도 이상 수직으로 회전하며 루프를 도는 우투기(비행 시뮬레이터)나 무중력 상태를 구현하려면 `Quaternion(사원수)` 수학 기반의 카메라 체계를 새로 짜야 짐벌 락(Gimbal Lock) 한계를 우회할 수 있다.
