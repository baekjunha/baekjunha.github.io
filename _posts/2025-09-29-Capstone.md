## 1. AI 코어 아키텍처: 지능 및 메모리 시스템

본 섹션은 AI 캐릭터의 인지 능력, 정체성 유지 및 플레이어 데이터 상호작용을 위한 핵심 기술을 분석합니다.

*   **LLM (Gemini) API 통신**: 플레이어 입력을 AI 서버로 전송하고 응답을 수신하는 주요 통신 채널입니다. 실시간 대화 생성을 수행합니다.
*   **RAG (Retrieval-Augmented Generation) 시스템**: LLM의 정보 처리 한계를 보완하는 기술입니다. 캐릭터의 설정, 배경 시나리오 및 플레이어와의 상호작용 데이터를 데이터베이스에 저장(Long-term Memory)합니다. 대화 맥락에 적합한 정보를 실시간으로 검색하여 LLM에 제공함으로써 응답의 일관성을 확보합니다.
    *   본 프로젝트는 **하이브리드 검색(Hybrid Search)** 방식을 적용합니다. 이는 단순 의미 기반 검색을 넘어, 호감도나 관계 상태 등의 메타데이터를 우선 필터링하는 방식입니다. 이를 통해 과거의 대화 맥락과 감정 상태를 반영한 정밀한 응답 생성이 가능합니다.
*   **프롬프트 엔지니어링**: AI에 캐릭터의 페르소나, 어조, 행동 규칙을 정의하는 과정입니다. 체계적인 프롬프트 설계는 응답의 일관성을 강화하며 토큰 사용량을 최적화하여 API 비용을 절감합니다.

## 2. 클라이언트 구현: 아키텍처 및 비용 최적화

본 섹션은 AI 모델과의 안정적인 통신, 사용자 경험 품질 확보 및 운영 비용 최적화를 위한 기술 구성을 다룹니다.

*   **UniTask 기반 비동기 처리**: LLM API 응답 대기 시 발생하는 메인 스레드 정지 현상을 방지합니다. 비동기 방식을 통해 안정적인 프레임워크 성능을 유지합니다.
*   **네트워크 가용성 설계**: API 응답 지연에 대비한 **타임아웃(Timeout)** 처리 및 통신 오류 시의 **재시도(Smart Retry)** 로직을 구현합니다.
*   **의존성 주입 (VContainer)**: 통신, 데이터 관리, UI 등 각 모듈을 독립적으로 분리하는 패턴입니다. 결합도를 낮춰 유지보수성을 향상시키고 효율적인 협업 구조를 구축합니다.
*   **응답 캐싱 시스템**: 정형화된 질의에 대해 로컬 캐시 응답을 반환하여 API 호출 비용을 절감하고 응답 속도를 향상시킵니다.
*   **모델 티어링 (Model Tiering)**: 대화의 중요도에 따라 경량 모델과 고성능 모델을 동적으로 전환하여 비용 효율성을 극대화합니다.

## 3. 게임플레이 시스템: 데이터 및 상호작용 로직

플레이어의 행동이 캐릭터와의 관계 데이터에 반영되도록 구조화하는 로직 및 데이터 설계 기술입니다.

*   **ScriptableObject 기반 데이터 관리**: 호감도, 신뢰도 등의 스탯을 에셋 단위로 관리하여 기획 데이터의 독립성을 확보하고 밸런스 튜닝 효율을 높입니다.
*   **관계 발전 시스템**: 플레이어의 선택이 호감도 데이터에 실시간 반영되는 시스템입니다. 수치 변화 및 상태 UI를 통해 사용자에게 피드백을 전달합니다.
*   **하이브리드 데이터베이스**: 로컬 데이터는 **SQLite**에 저장하고, 동기화가 필요한 계정 데이터는 **Firebase**를 활용하는 이원화된 데이터 관리 방식을 적용합니다.

## 4. UX 및 라이브 운영: 시각화 및 자동화 인프라

캐릭터 시각화 및 배포 후 데이터 기반의 품질 개선을 위한 기반 기술입니다.

*   **동적 립싱크 시스템**: LLM 응답을 TTS로 변환하고, **uLipSync**를 통해 오디오 파형과 3D 아바타의 **입 모양(Blend Shape)**을 실시간 동기화하여 시각적 몰입감을 확보합니다.
*   **CI/CD 파이프라인**: **GitHub Actions**를 활용하여 빌드 및 배포 프로세스를 자동화함으로써 제품 릴리즈 안정성을 보장합니다.
*   **데이터 피드백 루프**: **Firebase Analytics**를 통해 응답 품질에 대한 사용자 데이터를 수집하고, 이를 바탕으로 RAG 데이터셋 고도화 및 프롬프트 파라미터 미세조정을 수행합니다.
