## 1. AI 코어 아키텍처: 캐릭터의 지능과 기억

이 파트는 AI 캐릭터가 지능적으로 사고하고, 일관된 정체성을 유지하며, 플레이어와의 관계를 기억하게 만드는 핵심 기술을 다룬다.

*   **LLM (Gemini) API 통신**: 플레이어의 입력을 AI 서버로 전송하고 응답을 받아오는 핵심 통신 채널이다. 게임 내 실시간 대화 생성을 전담한다.
*   **RAG (검색 증강 생성) 시스템**: LLM의 기억력 한계를 보완하는 기술이다. 캐릭터의 고유한 성격, 배경 이야기, 플레이어와의 주요 사건 등 **장기 기억**을 데이터베이스에 저장한다. 이후 대화의 맥락에 맞는 정보를 실시간으로 검색하여 LLM에게 제공함으로써, 캐릭터가 일관된 정체성을 유지하며 답변하도록 돕는다.
    *   특히 본 프로젝트는 **하이브리드 검색(Hybrid Search)** 방식을 채택한다. 이는 단순히 사용자의 질문과 의미적으로 유사한 기억을 찾는 것을 넘어, **'호감도'나 '관계 상태' 같은 현재 게임 데이터와 연결된 정보(메타데이터)를 먼저 필터링**하는 고도화된 기술이다. 이로 인해 AI는 '어제 다퉜던 상황'을 인지하고 맥락에 맞는 미묘한 감정적 답변을 생성할 수 있다.
*   **프롬프트 엔지니어링**: AI에게 캐릭터의 정체성, 말투, 행동 규칙을 명시적으로 지시하는 과정이다. 체계적인 프롬프트 설계는 캐릭터의 일관성을 강화하며, 불필요한 토큰 낭비를 막아 **API 호출 비용을 직접적으로 절감**한다.

## 2. 클라이언트 구현: 성능, 안정성, 비용 관리

이 파트는 AI 모델과의 통신을 안정적으로 처리하고, 사용자에게 쾌적한 경험을 제공하며, 장기적인 운영 비용을 최적화하는 기술들을 구성한다.

*   **UniTask 기반 비동기 처리**: LLM API의 응답을 기다리는 동안 게임 프레임이 멈추는 프리징 현상을 방지한다. 모든 네트워크 통신을 메인 스레드와 분리된 비동기 방식으로 처리하여 **우수한 체감 성능을 제공**한다.
*   **네트워크 복원력 설계**: API 응답이 지연될 경우를 대비한 **타임아웃(Timeout)** 처리와 일시적인 통신 오류 시 서버 리소스를 낭비하지 않는 **스마트 재시도(Smart Retry)** 로직을 구현한다.
*   **의존성 주입 (VContainer)**: API 통신, 데이터 관리, UI 등 각 기능 모듈을 독립적으로 분리하는 아키텍처 패턴이다. 기능 간의 결합도를 낮춰 코드의 **유지보수성과 확장성을 비약적으로 높이고, 팀원 간 병렬 협업을 가능케 한다**.
*   **지능형 응답 캐싱**: "이름이 뭐야?"처럼 상황과 무관하게 답변이 동일한 기계적인 질의는 응답을 로컬 캐시에 저장한다. 중복된 API 호출을 차단해 **비용을 절감하고 즉각적인 응답 속도를 보장**한다.
*   **모델 티어링**: 간단한 스몰토크에는 가볍고 저렴한 소형 AI 모델을, 관계 분기점이 되는 핵심 대화에는 고성능 모델을 스위칭하는 등 **상황의 가중치에 따라 모델 체급을 동적으로 할당**하여 비용 효율을 극대화한다.

## 3. 게임플레이 시스템: 데이터 관리와 상호작용

이 파트는 플레이어의 행동이 AI 캐릭터와의 관계망에 실질적인 영향을 끼치도록 구조화하는 게임 로직 및 데이터 설계 기술이다.

*   **ScriptableObject 기반 데이터 설계**: **호감도, 신뢰도** 같은 캐릭터의 스테이터스 지표를 하드코딩과 분리된 에셋 파일 단위로 관리한다. 이를 통해 기획 파트가 코드를 건드리지 않고도 **게임 밸런스를 즉각적으로 튜닝**할 수 있는 환경을 제공한다.
*   **관계 발전 시스템**: 플레이어의 다이얼로그 선택이 캐릭터의 호감도 데이터에 실시간으로 반영되는 시스템이다. **"+10 호감도"** 팝업과 **관계 진행 막대(Progress Bar)** UI를 띄워 플레이어 행동에 대한 명확한 인과적 피드백을 전달한다.
*   **하이브리드 데이터베이스**: 대화 기록처럼 기기 내 보관으로 충분한 휘발성 로컬 데이터는 **SQLite**에 보관하고, 여러 디바이스에서 동기화가 필수적인 유저 계정 데이터는 **Firebase(클라우드 DB)** 에 적재하는 하이브리드 투트랙 방식을 취한다.

## 4. UX 및 라이브 운영: 몰입감과 지속 가능한 성장

이 파트는 AI 캐릭터에게 실감 나는 생명력을 부여하고, 배포 이후 유저 데이터를 바탕으로 지속가능한 퀄리티업을 수행하는 기반 기술들이다.

*   **동적 립싱크 시스템**: LLM이 산출한 텍스트 뭉치를 음성(TTS) 합성으로 변환하고, **uLipSync** 플러그인으로 음향 파형을 실시간 분석하여 3D 아바타의 **입 모양(Blend Shape)을 동기화**한다. 버추얼 휴먼과 실제로 교감하는 듯한 강렬한 **몰입감을 선사**하는 킬러 UX 지표다.
*   **CI/CD 파이프라인**: **GitHub Actions** 명세서들을 구축해, 코드 변경사항이 Main 브랜치에 Merge 될 때마다 **빌드와 배포 프로세스를 완전 자동화**한다. 이를 기반으로 치명적 버그 핫픽스나 시즌 콘텐츠 릴리즈를 신속하고 안정적으로 수행한다.
*   **AI 피드백 루프**: **Firebase Analytics**를 연동하여 AI 답변 퀄리티에 대한 유저 리액션(좋아요/싫어요)의 정량 데이터를 수집한다. 이 지표를 근거로 RAG 코퍼스를 고도화하거나 프롬프트 파라미터를 미세조정하여 **AI 캐릭터 모델이 스스로 성장하고 진화**하는 자가 발전 운영 사이클을 완성한다.

## 5. 프로젝트 기술 스택 학습 가이드

각 파트별 선별된 기술 스택을 효율적으로 온보딩하기 위한 아카이브다. 공식 문서를 뼈대 삼아, 튜토리얼 실습으로 살을 붙이고, 커뮤니티 베스트 프랙티스를 벤치마킹하는 3단계 학습을 권장한다.

### 5.1. AI 코어 아키텍처

*   **Gemini API**
    *   **공식 문서**: [Google AI for Developers](https://ai.google.dev/docs) (가장 빈틈없는 레퍼런스)
    *   **빠른 시작 가이드**: [Gemini API C# Quickstart](https://www.google.com/search?q=https://ai.google.dev/docs/csharp_quickstart) (C# 닷넷 환경 기반 API 바인딩 첫걸음)
*   **RAG 및 프롬프트 엔지니어링**
    *   **개념 학습**: [Google Cloud RAG Use case](https://cloud.google.com/use-cases/retrieval-augmented-generation?hl=ko) (RAG 패러다임에 대한 비즈니스 이해)
    *   **온라인 강의**: [DeepLearning.AI RAG Course](https://www.deeplearning.ai/short-courses/langchain-for-llm-application-development/) (LangChain 프레임워크를 투입한 RAG 벡터 구축 심화 학습)
    *   **프롬프트 가이드**: [Google Prompt Design Guide](https://www.google.com/search?q=https://cloud.google.com/vertex-ai/docs/generative-ai/learn/prompt-design)

### 5.2. 클라이언트 구현

*   **Unity C# 기초**
    *   **공식 학습 플랫폼**: [Unity Learn](https://learn.unity.com/) (게임 엔진 철학과 C# 문법의 결합점 마스터)

*   **UniTask (비동기 처리)**
    *   **공식 GitHub**: [Cysharp/UniTask](https://github.com/Cysharp/UniTask) (유니티에 최적화된 Zero-Allocation async/await 메커니즘 튜토리얼)
*   **VContainer (의존성 주입)**
    *   **공식 문서**: [VContainer Docs](https://vcontainer.hadashikick.jp/) (DI 컨테이너 구현체 적용법)
    *   **튜토리얼 영상**: [Unity VContainer Basics](https://www.youtube.com/watch?v=17U3bLkFgEU) (제어의 역전(IoC) 개념 시각화 강좌)

### 5.3. 게임플레이 시스템

*   **ScriptableObject**
    *   **공식 매뉴얼**: [Unity ScriptableObject Manual](https://docs.unity3d.com/kr/current/Manual/class-ScriptableObject.html)
