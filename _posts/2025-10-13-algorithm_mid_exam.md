## 1. 분할 정복 알고리즘 (Divide and Conquer)

문제를 가장 작은 단위로 쪼갠 뒤, 아래에서부터 해를 결합해 올라가는 탑다운(Top-down) 방식의 전략이다.

### 1.1 합병 정렬 (Merge Sort)

"일단 절반으로 나누고, 합치면서 정렬한다"는 원리를 따른다.

#### 동작 구조
1.  **분할(Divide)**: 배열을 중간 지점 `k`를 기준으로 두 개의 부분 배열로 나눈다.
2.  **정복(Conquer)**: 왼쪽과 오른쪽 부분 배열을 재귀적으로 정렬한다.
3.  **결합(Combine)**: 정렬된 두 부분 배열을 `Merge` 함수를 통해 하나의 정렬된 배열로 합병한다.

```c++
void MergeSort(int A[], int p, int q) {
    if (p < q) {
        int k = (p + q) / 2;
        MergeSort(A, p, k);
        MergeSort(A, k + 1, q);
        Merge(A, p, k, q);
    }
}
```
{: file="MergeSort.cpp" }

#### 시간 복잡도 유도
*   **분할 깊이**: 크기 $n$의 배열이 1이 될 때까지 2로 나누므로, 깊이는 $k = \log_2 n$ 이다.
*   **층별 연산**: 각 층(Level)에서 수행되는 합병 연산의 총합은 항상 $n$개의 원소를 비교하므로 $O(n)$ 이다.
*   **최종 복잡도**: (깊이) $\times$ (층별 연산) = $O(n \log n)$ 이다. 이는 데이터의 분포와 무관하게 항상 일정하다.

### 1.2 퀵 정렬 (Quick Sort)

피봇(Pivot)을 기준으로 배열을 파티셔닝하며 정렬을 수행한다.

#### 시간 복잡도 유도
*   **최선/평균**: 피봇이 항상 중앙에 가깝게 위치하여 배열을 균등하게 이분할하는 경우다.
    *   분할 깊이: $O(\log n)$
    *   비교 횟수: 각 층에서 $O(n)$
    *   결과: **$O(n \log n)$**
*   **최악**: 이미 정렬된 배열에서 맨 앞이나 맨 뒤를 피봇으로 잡아 배열이 $n-1$과 $0$으로 비균등하게 나뉘는 경우다.
    *   분할 깊이: $O(n)$
    *   결과: **$O(n^2)$**

### 1.3 선택 문제 (Selection Problem)

$n$개의 숫자 중 $k$번째로 작은 수를 찾는 알고리즘으로, 퀵 정렬의 `Partition` 개념을 활용한다.

#### 동작 원리
피봇 위치 $p$를 기준으로 $k$값이 있는 쪽만 한정하여 재귀 호출을 수행한다. 퀵 정렬과 달리 한쪽 방향만 탐색하므로 평균적으로 더 빠르다.

#### 시간 복잡도 유도
*   **평균**: 문제의 크기가 기하급수적으로 감소한다 ($n \rightarrow \frac{3}{4}n \rightarrow \dots$). 총 연산량은 등비수열의 합($n + \frac{3}{4}n + \dots$)에 수렴하여 **$O(n)$**이 된다.
*   **최악**: 퀵 정렬의 최악의 경우와 동일하게 **$O(n^2)$**이 발생한다.

### 1.4 최근접 점의 쌍 찾기 (Closest Pair)

2차원 평면의 $n$개 점 중 최소 거리를 가진 쌍을 찾는다.

#### 시간 복잡도 유도
재귀식은 $T(n) = 2T(n/2) + O(n \log n)$ 이다. 여기서 $O(n \log n)$은 중간 영역(Strip)의 점들을 $y$좌표 기준으로 정렬하는 데 소요되는 시간이다.
이를 풀면 최종 복잡도는 **$O(n \log^2 n)$**이 되며, 정렬 최적화 시 **$O(n \log n)$**까지 개선 가능하다.

## 2. 그리디 알고리즘 (Greedy Algorithm)

"현재 상황에서 가장 최적인 것"을 선택하여 해를 구하는 방식이다. 모든 경우에 최적해를 보장하지는 않으나, 특정 조건을 만족하는 문제에서는 매우 효율적이다.

### 2.1 동전 거스름돈 (Coin Change)

가장 큰 액면가의 동전부터 우선적으로 선택한다. 액면가 간의 배수 관계가 성립할 때 최적해를 보장한다.

*   **시간 복잡도**: 동전의 종류를 $k$라 할 때, 단순히 루프를 돌면 **$O(k)$** 또는 액수 $W$에 비례하는 **$O(W)$**로 분석된다.

### 2.2 최소 신장 트리 (MST)

#### 2.2.1 크러스컬 알고리즘 (Kruskal)
가장 가중치가 낮은 간선부터 사이클을 형성하지 않는 범위 내에서 선택한다.

*   **동작 원리**: 간선을 가중치 순으로 정렬한 뒤, Union-Find 자료구조를 사용하여 사이클 발생 여부를 점검한다.
*   **시간 복잡도**: 간선 정렬 비용이 지배적이다. 간선 개수가 $m$일 때 **$O(m \log m)$** 이다.

#### 2.2.2 프림 알고리즘 (Prim)
하나의 정점에서 시작하여 연결된 간선 중 가장 가중치가 낮은 것을 선택하며 트리를 확장한다.

*   **시간 복잡도**:
    *   배열 기반 구현: **$O(n^2)$**
    *   우선순위 큐(Heap) 기반: **$O(m \log n)$**

### 2.3 최단 경로 (Dijkstra)

특정 출발점에서 모든 정점까지의 최단 거리를 구한다. 프림 알고리즘과 유사하나, 누적 거리를 기준으로 간선 완화(Relaxation)를 수행한다는 차이가 있다.

*   **시간 복잡도**: 배열 기반 구현 시 **$O(n^2)$** 이 소요된다.

### 2.4 부분 배낭 문제 (Fractional Knapsack)

물건을 쪼갤 수 있을 때, "단위 무게당 가치"가 높은 순으로 담는다.

*   **시간 복잡도**: 물건 정렬 시간이 핵심이며 **$O(n \log n)$** 이 소요된다.

### 2.5 집합 커버 문제 (Set Cover)

아직 커버되지 않은 원소를 가장 많이 포함하는 집합을 반복적으로 선택한다. 근사해를 찾는 그리디 전략의 대표 사례다.

*   **시간 복잡도**: 집합과 원소의 개수를 고려할 때 일반적으로 **$O(n^3)$** 정도의 다항 시간이 소요된다.

### 2.6 작업 스케줄링 (Job Scheduling)

시작 시간이 가장 빠른 작업부터 배정한다.

*   **시간 복잡도**: 작업 정렬($O(n \log n)$) 후 기계 배정 루프($O(nm)$)를 거치므로 **$O(n \log n + nm)$** 이 소요된다.

### 2.7 허프만 압축 (Huffman Coding)

빈도수가 낮은 두 노드를 묶어 트리를 구성한다.

*   **시간 복잡도**: 우선순위 큐 연산을 $n$번 반복하므로 **$O(n \log n)$** 이 소요된다.

## Chapter 5: 동적 계획 알고리즘 (Dynamic Programming Algorithm) 

동적 계획법은 큰 문제를 작은 문제로 나누어 푼다는 점에서 분할 정복과 비슷하지만, 중복되는 계산을 피하기 위해 작은 문제들의 답을 저장해두고 재활용하는 점이 핵심입니다. 복잡한 최적화 문제를 해결하는 데 매우 유용합니다.

* 5.1 모든 쌍 최단 경로 (All Pairs Shortest Paths)
* 5.2 연속 행렬 곱셈 (Chained Matrix Multiplication)
## 3. 동적 계획법 (Dynamic Programming)

최적 부분 구조(Optimal Substructure)와 중복되는 부분 문제(Overlapping Subproblems) 성질을 가진 문제를 해결하는 방식이다. 이미 계산된 해를 테이블에 저장(Memoization)하여 중복 연산을 방지한다.

### 3.1 모든 쌍 최단 경로 (Floyd-Warshall)

모든 정점 쌍 사이의 최단 거리를 구한다. 특정 정점 $k$를 경유지로 거쳐가는 경우를 고려하여 최단 거리를 갱신한다.

*   **동작 원리**: $D[i,j] = \min(D[i,j], D[i,k] + D[k,j])$ 점화식을 사용한다.
*   **시간 복잡도**: 3중 중첩 루프가 수행되므로 **$O(n^3)$** 이 소요된다.

### 3.2 연속 행렬 곱셈 (Chained Matrix Multiplication)

$n$개의 행렬을 곱할 때, 최소 연산 횟수를 보장하는 곱셈 순서를 찾는다.

*   **시간 복잡도**: 부분 행렬의 범위와 분할 지점을 모두 탐색하므로 3중 루프 구조를 가지며, **$O(n^3)$** 이 소요된다.

### 3.3 편집 거리 (Edit Distance)

문자열 $S$를 $T$로 변환하는 데 필요한 최소 삽입, 삭제, 대체 연산 횟수를 구한다.

*   **시간 복잡도**: 두 문자열의 길이를 각각 $m, n$이라 할 때, $m \times n$ 테이블을 채우므로 **$O(mn)$** 이 소요된다.

### 3.4 0-1 배낭 문제 (Knapsack Problem)

물건을 쪼갤 수 없을 때, 배낭의 용량 $C$ 내에서 최대 가치를 찾는다.

*   **시간 복잡도**: 물건의 개수 $n$과 배낭 용량 $C$에 대해 테이블을 채우므로 **$O(nC)$** 이 소요된다. 용량 $C$가 상수가 아니므로 의사 다항 시간(Pseudo-polynomial time) 복잡도를 가진다.

### 3.5 동전 거스름돈 (Coin Change - DP)

그리디 방식이 최적해를 보장하지 못할 때 사용한다.

*   **시간 복잡도**: 거스름돈 액수 $n$과 동전 종류 $k$에 대해 **$O(nk)$** 이 소요된다.

## 4. 결론 및 요약

알고리즘 설계 기법은 문제의 성격에 따라 선택되어야 한다.

| 기법 | 핵심 전략 | 대표 사례 | 주요 복잡도 |
| :--- | :--- | :--- | :--- |
| **분할 정복** | 문제를 쪼개서 정복 후 결합 | 합병 정렬, 퀵 정렬 | $O(n \log n)$ |
| **그리디** | 매 순간 최적의 선택 | MST(Prim, Kruskal), 다익스트라 | 다양 ($O(m \log n)$ 등) |
| **동적 계획법** | 부분 해의 재사용 | 플로이드-워셜, 배낭 문제 | 다양 ($O(n^3), O(nC)$ 등) |

각 알고리즘의 시간 복잡도 유도 과정을 이해하는 것은 효율적인 프로그래밍의 기초가 된다. 특히 재귀적 구조를 반복문과 Memoization으로 변환하는 DP의 사고방식은 복잡한 문제 해결에 필수적이다.
