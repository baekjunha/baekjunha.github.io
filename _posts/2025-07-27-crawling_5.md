## 2. 기본 파싱 패턴과 한계점

이전 섹션에서 배운 `find_all` 메서드를 기반으로 한 필터링 로직을 단계별로 고도화해보자.

### 2.1. 전체 목록 무지성 추출 (1단계)

특정 `class`만 던져주면 문서 내의 모든 동명이인 태그가 몽땅 딸려온다.

```python
import requests
from bs4 import BeautifulSoup

res = requests.get('https://davelee-fun.github.io/blog/crawl_test')
soup = BeautifulSoup(res.content, 'html.parser')

# 'course' 클래스를 가진 모든 <li> 태그 색인
titles = soup.find_all('li', 'course') 
for title in titles:
    print(title.get_text())
```

출력을 보면 취미 코딩과 개발자 코너가 짬뽕되어 나온다.

```text
1. 취미코딩-파이썬입문[무료강의]
2. 취미코딩-데이터분석[무료강의]
...
```

### 2.2. 특정 ID로 구역 한정 (2단계)

특정 부모 박스를 먼저 족집게로 집어낸 뒤, 그 체인 안에서만 자식을 수색하는 이중 필터링을 적용한다. 훨씬 안전하고 정확한 방식이다.

```python
# 1. 취미 강좌를 감싸는 <ul> 컨테이너 타겟팅
section = soup.find('ul', id='hobby_course_list')

# 2. 해당 컨테이너 내부에서만 <li class="course"> 색인
titles = section.find_all('li', 'course')
for title in titles:
    print(title.get_text())
```
{: file="crawler.py" }

원하는 취미(hobby) 강좌 라인업만 깔끔하게 도출된다.

### 2.3. 문자열 전처리 및 인덱싱 (3단계)

데이터를 긁어오는 것만큼 예쁘게 정제하는 것도 중요하다.

```python
section = soup.find('ul', id='dev_course_list')
titles = section.find_all('li', 'course')

for index, title in enumerate(titles):
    # '-'로 쪼개고, '['로 한 번 더 쪼개어 알맹이만 파싱
    raw_text = title.get_text()
    refined_text = raw_text.split('-')[-1].split('[')[0].strip()
    
    print(str(index + 1) + '.', refined_text)
```
{: file="crawler.py" }

**문자열 쪼개기(Parsing) 단계:**
```text
원본: "3. 개발자되기-웹프로그래밍[중급]"
    ↓ .split('-')[-1]
"웹프로그래밍[중급]"
    ↓ .split('[')[0]
"웹프로그래밍"
    ↓ .strip()
"웹프로그래밍"
```

## 4. BeautifulSoup 잡학 사전 (Tips)

### 4.1. 단일과 복수 탐색
*   **`find()`**: 트리에서 조건에 맞는 **첫 번째 단일 노드**만 찾고 즉시 순회를 종료한다. 
*   **`find_all()`**: 끝까지 뒤져서 조건에 맞는 모든 노드를 긁어모아 **리스트(List) 컬렉션**으로 반환한다.

### 4.2. 명시적 속성 타겟팅
클래스명으로 찾을 때 두 가지 방식이 있다.

```python
soup.find_all('li', 'course')         # 암묵적 (클래스라고 추측함)
soup.find_all('li', class_='course')  # 명시적 (파이썬의 마법 예약어 피하기 위해 언더바 _ 사용)
```

## 5. 끝판왕 타겟팅: CSS Selector 

`find_all`로 복잡한 DOM 트리를 계단식으로 파고드는 건 코드가 지저분해진다. `BeautifulSoup`은 프론트엔드 개발자들이 스타일을 먹일 때 쓰는 **CSS 문법을 역이용하여 단 단 한 줄로 노드를 타겟팅**하는 `.select()` 강력한 무기를 제공한다.

### 5.1. 구문 종류
*   **`select('구문')`**: 구문에 매칭되는 모든 노드들을 리스트로 싹쓸이.
*   **`select_one('구문')`**: 구문에 매칭되는 첫 번째 단일 노드만 리턴.

---

## 6. CSS Selector 실전 핸드북

### 6.1. 기본 태그 싹쓸이
```python
items = soup.select('li')
```
HTML 문서의 모든 `<li>` 태그를 잡아온다. 너무 광범위해서 실무에선 잘 안 쓴다.

### 6.2. 하위 트리를 관통하는 띄어쓰기 (Descendant)
```python
items = soup.select('html body h1')
```
띄어쓰기는 "모든 하위 자손"을 뜻한다. 즉, `html` 안의 `body` 안의 어딘가 깊숙히 박혀 있는 `h1` 태그를 모두 찾아낸다.

### 6.3. 직계 자손만 매칭하는 꺾쇠 `>` (Child)
```python
items = soup.select('ul > li')
```
바로 밑에 태어난 1촌 직계 자식 `<li>`만 물어온다.

### 6.4. 클래스는 점 `.`, 아이디는 샵 `#`
```python
items = soup.select('.course')  # class="course"
items = soup.select('#start')   # id="start"
```
가장 많이 쓰게 될 핵심 식별자 기호다.

### 6.5. 찰진 속성 조합 기법
```python
items = soup.select('li.course.paid')
```
띄어쓰기 없이 속성을 바싹 붙여 쓰면 **AND 조건**이다. `<li>` 태그인데 덤으로 `course`와 `paid` 클래스를 동시에 두른 녀석들만 색출한다.

### 6.6. 계층과 속성의 혼종 (최다 빈도)
```python
items = soup.select('ul#hobby_course_list li.course')
```
"아이디가 `hobby_course_list`인 `<ul>` 박스를 먼저 찾고, 그 하위의 어떤 깊이든 클래스가 `course`인 `<li>` 태그를 다 가져와!" 라는 복합 명령을 띄어쓰기 하나로 압축해 낸다.

### 6.7. 스나이퍼 모드 단일 추출
```python
item = soup.select_one('ul#dev_course_list > li.course.paid')
```
이런 복잡한 식별 기호를 다 붙인 타겟의 **첫 번째 타겟만** `select_one`으로 아주 정확히 저격할 수 있다.

## 7. find vs select, 뭐가 더 좋을까?

복잡한 표 단위 행렬 데이터를 쪼개는 과정을 보면 답이 나온다.

```python
# select 기법을 사용한 아름다운 2중 루프 구성
items = soup.select('tr')
for item in items:
    columns = item.select('td') # 행 안에서 열들만 다시 select
    row_str = ''
    for column in columns:
        row_str += ', ' + column.get_text()
    print(row_str[2:]) # 앞 쉼표 자르기
```

`select()` 계열 함수는 `find_all()`보다 사용 문법이 훨씬 압축적이며, CSS 계층 트리를 그대로 가져와서 복사-붙여넣기 할 수 있다는(예: Chrome 개발자 도구 - Copy Selector) 치명적인 장점이 있어, 실무 대다수의 스크래퍼들이 `.select()`를 주력으로 채택한다.

## 8. CSS Selector 치트시트

| 문법 | 타겟팅 로직 (의미) | 예시 (Example) |
| :--- | :--- | :--- |
| `'tag'` | 모든 해당 태그명 | `'li'` |
| `'.class'` | 해당 클래스 네임택 | `'.course'` |
| `'#id'` | 단일 고유 ID | `'#start'` |
| `'A > B'` | A의 직계 1촌 자식 B | `'ul > li'` |
| `'A.class'`| 여러 속성 동시 조건 | `'li.course.paid'` |
| `'A#id B.c'`| 부모/자손 조합 검색 | `'ul#hobby_course_list li.course'` |
