# 11. 자동 스케줄러 가이드 (macOS Scheduler Setup Guide)

이 문서는 macOS 환경에서 `실시간_뉴스페이퍼_갱신.sh` 스크립트를 주기적으로 자동 실행하기 위한 백그라운드 스케줄러(cron 및 launchd) 등록 방법을 설명합니다. 

맥북의 상태와 선호도에 맞춰 아래 두 가지 방법 중 하나를 선택해 구성할 수 있습니다.

---

## 방법 1. 간단하고 직관적인 크론탭 (cron) 설정

크론탭은 리눅스 및 macOS에서 가장 널리 쓰이는 주기적 작업 예약 도구입니다.

### 1. 설정 방법
1. 터미널 앱을 실행합니다.
2. 아래 명령어를 입력하여 크론탭 편집기를 엽니다.
   ```bash
   crontab -e
   ```
3. 키보드의 `i` 키를 눌러 입력 모드(Insert Mode)로 전환한 뒤, 아래 설정 행을 복사해 붙여넣습니다. (아래 경로는 사용자 홈 디렉토리를 기준으로 설계되었습니다.)
   ```text
   0 9,18 * * * cd /Users/baekjunha/baekjunha.github.io && ./실시간_뉴스페이퍼_갱신.sh > /dev/null 2>&1
   ```
   * **주기 설명**: 매일 오전 9시 정각 및 오후 6시 정각에 뉴스페이퍼를 자동으로 수집하여 마크다운 파일로 저장합니다.
4. 붙여넣기가 완료되면 `Esc` 키를 누르고, `:wq`를 입력한 뒤 `Enter` 키를 눌러 저장하고 빠져나옵니다.

### 2. macOS 권한 부여 (필수)
macOS 모하비(Mojave) 이상 버전에서는 크론탭이 디스크 파일에 정상적으로 접근하기 위해 터미널 및 크론 바이너리에 전체 디스크 접근 권한을 요구하는 경우가 있습니다.
1. **시스템 설정** -> **개인정보 보호 및 보안** -> **전체 디스크 접근 권한**으로 이동합니다.
2. **터미널(Terminal)** 및 시스템 내 `/usr/sbin/cron`을 활성화 목록에 추가합니다.

---

## 방법 2. macOS 네이티브 스케줄러 (launchd) 설정 (추천)

`launchd`는 macOS의 공식 데몬 및 에이전트 관리 도구입니다. 

> [!TIP]
> **launchd의 장점**
> 크론(cron)은 지정된 실행 시간에 맥북이 잠자기(Sleep) 상태일 경우 해당 타이밍을 그냥 건너뛰어 버립니다. 반면, **launchd**는 맥북이 깨어나는 즉시 밀렸던 누적 스케줄을 감지하여 자동으로 뉴스 수집기를 한 번 실행해 주는 강점이 있습니다.

### 1. 에이전트 설정 파일 (plist) 생성
1. 터미널을 열고 에이전트 설정 경로로 이동합니다.
   ```bash
   cd ~/Library/LaunchAgents
   ```
2. 아래 이름의 파일을 새로 생성합니다.
   ```bash
   touch com.user.jobsearchos.newspaper.plist
   ```
3. 에디터로 이 파일을 열어 아래 XML 소스 코드를 붙여넣습니다. (경로 부분을 본인의 맥북 절대 경로로 올바르게 확인하세요.)
   ```xml
   <?xml version="1.0" encoding="UTF-8"?>
   <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
   <plist version="1.0">
   <dict>
       <key>Label</key>
       <string>com.user.jobsearchos.newspaper</string>
       
       <key>ProgramArguments</key>
       <array>
           <string>/bin/bash</string>
           <string>/Users/baekjunha/baekjunha.github.io/실시간_뉴스페이퍼_갱신.sh</string>
       </array>
       
       <key>StartCalendarInterval</key>
       <array>
           <!-- 매일 오전 9시 00분 실행 -->
           <dict>
               <key>Hour</key>
               <integer>9</integer>
               <key>Minute</key>
               <integer>0</integer>
           </dict>
           <!-- 매일 오후 6시 00분 실행 -->
           <dict>
               <key>Hour</key>
               <integer>18</integer>
               <key>Minute</key>
               <integer>0</integer>
           </dict>
       </array>
       
       <!-- 사용자 로그인 시(부팅 시) 최초 1회 즉시 실행 여부 -->
       <key>RunAtLoad</key>
       <true/>
       
       <key>StandardOutPath</key>
       <string>/Users/baekjunha/baekjunha.github.io/Job_Search_OS/11_모니터링/newspaper_stdout.log</string>
       <key>StandardErrorPath</key>
       <string>/Users/baekjunha/baekjunha.github.io/Job_Search_OS/11_모니터링/newspaper_stderr.log</string>
   </dict>
   </plist>
   ```

### 2. 스크립트 에이전트 등록 및 작동 명령어
파일 작성을 마쳤다면, 아래 터미널 명령어를 입력하여 macOS 시스템 에이전트에 스케줄 작업을 정상 등록 및 로드(Load)합니다.

* **에이전트 로드 (스케줄 가동 시작)**:
  ```bash
  launchctl load ~/Library/LaunchAgents/com.user.jobsearchos.newspaper.plist
  ```
* **수동으로 강제 즉시 실행 테스트**:
  ```bash
  launchctl start com.user.jobsearchos.newspaper
  ```
* **에이전트 언로드 (스케줄 중단)**:
  ```bash
  launchctl unload ~/Library/LaunchAgents/com.user.jobsearchos.newspaper.plist
  ```

## 3. 스케줄러 작동 여부 최종 검증 방법

1. 예약된 주기가 도래하거나 강제 시작 명령어를 준 뒤, `Job_Search_OS/11_모니터링/` 폴더 내부에 [일일_신문.md](file:///Users/baekjunha/baekjunha.github.io/Job_Search_OS/11_모니터링/일일_신문.md) 파일의 '최종 갱신일시' 메타데이터가 현재 시간으로 바뀌었는지 옵시디언 대시보드상에서 최종 점검합니다.
2. 실행에 실패한다면 `~/Library/LaunchAgents` 설정에 지정해 둔 출력 로그(`newspaper_stdout.log` 및 `newspaper_stderr.log`)를 확인하여 에러 원인을 추적합니다.
